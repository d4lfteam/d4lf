description: Write a short English maintainer-facing note for d4lf changes. Use when the user wants a concise PR note, changelog note, root-cause summary, or maintainer explanation after a fix.
---

Write a compact note in English with four parts when applicable:
1. Root cause
2. Code change
3. Cross-file or behavior impact
4. Validation

Rules:
- Keep it short and concrete.
- Do not oversell certainty.
- Mention files only when that adds useful context.
- Prefer plain technical wording over marketing language.
- If validation was limited, say so explicitly.
